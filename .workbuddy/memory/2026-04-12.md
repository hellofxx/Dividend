## 定投(DCA)策略逻辑修正 (22:04-22:12)

### 问题
定投收益率与一次性买入趋同——需简化为固定10000元/月的标准定投模式。

### 修复 (v3.3 → v3.4)
1. **models.py**: dca_monthly_amount 默认值 0.0 → **10000.0**
2. **benchmark.py `_run_dca()`** 完全重写：
   - 移除资金池(total_budget)、cash余额、min()扣款限制
   - 每月直接投入固定金额(默认10000)，无预存
   - 资金曲线仅含持仓市值(shares * nav)
   - 收益率基数=actual_invested(累计投入本金)，与init_cash完全脱钩
3. 文档同步: PRD/TDD/AGENTS 均升级至 v3.4

### DCA 曲线不显示/收益率错误 Bug (22:16-22:18)
- **Bug1**: DCA 资金曲线含 total_value=0 的行，compare.py 归一化时跳过整条曲线 → 过滤零值行
- **Bug2(核心)**: 资金曲线构建用循环结束后的最终 shares 计算所有日期 → **每天都当成已买完全部月份**
  - 修复: 引入 running_shares + buy_idx 逐日跟踪实际份额，当天买入日才累加
  - 效果: DCA 曲线呈阶梯式增长(每月投入后才增加份额)，符合真实定投逻辑

### 问题
定投策略收益率与一次性买入趋同——因为之前用 `init_cash` 作为收益率基数，而总投入恰好等于 init_cash。

### 修复 (benchmark.py + models.py)
1. **models.py**：StrategyConfig 新增 `dca_monthly_amount` 字段（=0 时自动按 init_cash/月数 分摊）
2. **benchmark.py `_run_dca()`** 完整重写：
   - 资金池 = `monthly_amount × 月数`（非 init_cash）
   - 累计 `invested` 变量跟踪实际已投入本金
   - 指标计算用 `max(invested, 1.0)` 作为 init_cash 参数
   - 结果：定投收益率天然不同于一次性买入
3. 文档同步更新：PRD v3.3 / TDD v3.3 / AGENTS.md 业务红线新增第6条
